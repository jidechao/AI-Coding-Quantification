# FAQ

## 这是不是在监控个人？

不是。默认看板聚合到团队或小组层级，目标是看 AI Coding 推广是否有效，而不是比较个人。

## 为什么不用 AI PR 数量做排名？

数量很容易被刷，也不能代表质量。方案更关注合入率、review 轮次、周期时间、缺陷和 churn。

## Human-Edit-Ratio 准吗？

V1 是启发式估算，会有误差。我们用抽样审计持续校准，不把单个 commit 的估算当作绝对事实。

## 如果我不确定标签怎么办？

选择更保守的一档。比如不确定是 `ai-authored` 还是 `ai-assisted`，就选 `ai-assisted`。

## 敏感代码可以不用 AI 吗？

可以。安全、密钥、合规和高风险模块可以标 `human`，并在 MR 中说明。

## 模型字段会不会泄露内部信息？

内部仓库可以保留具体模型；对外开源仓库建议统一写 `internal` 或 `none`。
