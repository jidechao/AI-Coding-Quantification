# 团队对齐：AI Coding 三档定义

本方案只用于观察团队采用 AI Coding 的过程和趋势，不用于个人绩效排名。

## 三档分类

| 标签 | 定义 | 参考标准 |
|---|---|---|
| `ai-authored` | AI 主导生成，人类主要做 review、微调和验收 | 人类修改占比约 0%-30% |
| `ai-assisted` | AI 起草或辅助，人类进行了明显改写和补充 | 人类修改占比约 30%-70% |
| `human` | 主要由人类完成，AI 仅用于补全、查询、解释或未实际生成代码 | 人类修改占比约 70%-100% |

## 为什么允许估算误差

V1 不读取 IDE 内部事件日志，`Human-Edit-Ratio` 由启发式算法估算。它会结合 commit message、diff 体量和代码风格信号，目标是形成团队趋势，不追求单个 commit 的绝对精确。

## 行为准则

- 不为了提高 AI 占比改变标签。
- 不用个人 AI PR 数排名。
- 标签不确定时优先选择较保守的一档。
- 敏感模块可以选择 `human` 并在 MR 描述说明原因。
- 每周抽样审计结果只用于修正流程和阈值。
